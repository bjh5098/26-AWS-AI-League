# 4.5 AWS Lambda — 치트시트

## TL;DR
Lambda = 에이전트 Tool의 실제 실행 환경. Python 코드 수정 → 배포 → 즉시 반영.

## Lambda Handler 기본 구조 (AgentCore용)
```python
import json

def lambda_handler(event, context):
    """AgentCore Tool Lambda 핸들러"""
    # 파라미터 추출
    action_group = event.get('actionGroup', '')
    function_name = event.get('function', '')
    params = {p['name']: p['value'] for p in event.get('parameters', [])}

    try:
        if function_name == 'find_path':
            result = handle_find_path(params)
        elif function_name == 'collect_coin':
            result = handle_collect_coin(params)
        else:
            result = {'error': f'Unknown function: {function_name}'}

        return build_response(action_group, function_name, result)

    except Exception as e:
        return build_response(action_group, function_name, {'error': str(e)})


def build_response(action_group, function_name, result):
    return {
        'response': {
            'actionGroup': action_group,
            'function': function_name,
            'functionResponse': {
                'responseBody': {
                    'TEXT': {'body': json.dumps(result, ensure_ascii=False)}
                }
            }
        }
    }
```

## 빠른 배포 (AWS CLI)
```bash
# 단일 파일 배포
zip function.zip lambda_function.py
aws lambda update-function-code \
  --function-name $LAMBDA_FUNCTION_NAME \
  --zip-file fileb://function.zip

# 배포 완료 대기
aws lambda wait function-updated --function-name $LAMBDA_FUNCTION_NAME
```

## 성능 최적화
| 설정 | 권장값 | 이유 |
|------|--------|------|
| 메모리 | 256MB | A* 알고리즘 실행에 충분 |
| 타임아웃 | 30초 | AgentCore 기본 대기시간 고려 |
| 동시성 | 기본값 | 대회 환경에서 충분 |


## 주요 에러
| 에러 | 원인 | 해결 |
|------|------|------|
| Task timed out | 알고리즘 실행 시간 초과 | 타임아웃 30초로 증가 |
| Runtime.ImportModuleError | 모듈 import 실패 | zip에 의존성 포함 확인 |
| MemorySize exceeded | 메모리 부족 | 512MB로 증가 |

## 대회 팁
- **표준 라이브러리만 사용**: numpy/scipy 설치 불확실 → 순수 Python으로 구현
- **에러 메시지 명확히**: 에이전트가 에러를 보고 다음 행동 결정 → 의미있는 메시지 필수
- **응답은 JSON**: `json.dumps(result, ensure_ascii=False)` 로 한글 포함 가능
