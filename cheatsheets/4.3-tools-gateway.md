# 4.3 Tools / Gateway — 치트시트

## TL;DR
Tools = 에이전트가 호출할 수 있는 외부 기능. Gateway = 여러 Tool을 묶어 단일 엔드포인트로 제공.

## Lambda Tool 등록 패턴

### Function Schema 방식 (단순, 권장)
```python
bedrock_agent.create_agent_action_group(
    agentId=agent_id,
    agentVersion='DRAFT',
    actionGroupName='CoinCollectorTools',
    actionGroupExecutor={'lambda': os.environ['LAMBDA_COIN_COLLECTOR_ARN']},
    functionSchema={
        'functions': [
            {
                'name': 'collect_coin',
                'description': '현재 위치의 코인을 수집합니다. 코인이 있을 때만 호출하세요.',
                'parameters': {
                    'position_x': {'type': 'integer', 'description': '현재 X 좌표', 'required': True},
                    'position_y': {'type': 'integer', 'description': '현재 Y 좌표', 'required': True},
                }
            }
        ]
    }
)
```

### Lambda Handler 패턴
```python
def lambda_handler(event, context):
    # AgentCore가 전달하는 파라미터 파싱
    action_group = event.get('actionGroup', '')
    function_name = event.get('function', '')
    parameters = {p['name']: p['value'] for p in event.get('parameters', [])}

    result = execute_tool(function_name, parameters)

    return {
        'response': {
            'actionGroup': action_group,
            'function': function_name,
            'functionResponse': {
                'responseBody': {
                    'TEXT': {'body': str(result)}
                }
            }
        }
    }
```

## Tool Description 품질이 핵심
에이전트가 "언제 이 도구를 쓸지" 판단하는 근거 = description
- ❌ 나쁜 예: `"경로를 찾습니다"`
- ✅ 좋은 예: `"현재 위치에서 목표 코인까지 장애물을 피한 최단 경로를 반환합니다. 이동 전에 반드시 호출하세요."`

## 주요 에러
| 에러 | 원인 | 해결 |
|------|------|------|
| DependencyFailedException | Lambda 실행 실패 | CloudWatch 로그 확인 |
| ActionGroupSignatureMismatch | 파라미터 타입 불일치 | schema의 type 재확인 |

## 대회 팁
- **Tool 수 최소화**: 3-5개 이내 권장 (너무 많으면 에이전트가 혼란)
- **명확한 description**: "언제 쓸지"를 명시적으로 작성
- **에러 시 재시도**: Lambda에 try/except + 의미있는 에러 메시지 반환
