# 4.2 AgentCore Memory — 치트시트

## TL;DR
Memory = 에이전트가 이전 맥락을 잊지 않고 다음 판단에 활용하는 기능. 힌트·코인위치·방문경로 저장.

## 핵심 전략
**매 턴 시작 시 Memory 조회** → 이미 방문한 곳 재탐색 방지 → 코인 수집 효율 극대화

## Key API Calls (boto3)

### Memory 생성
```python
import boto3, os

bedrock_agent = boto3.client('bedrock-agent', region_name=os.environ['AWS_REGION'])

memory = bedrock_agent.create_memory(
    name='ai-league-memory',
    memoryExecutionRoleArn=os.environ['AGENT_ROLE_ARN'],
)
memory_id = memory['memory']['memoryId']
```

### 에이전트에 Memory 연결
```python
bedrock_agent.update_agent(
    agentId=os.environ['AGENT_ID'],
    memoryConfiguration={
        'enabledMemoryTypes': ['SESSION_SUMMARY'],
        'storageDays': 1,
    }
)
```

### 런타임에서 Memory 조회
```python
runtime = boto3.client('bedrock-agent-runtime', region_name=os.environ['AWS_REGION'])

memories = runtime.get_agent_memory(
    agentId=os.environ['AGENT_ID'],
    agentAliasId=os.environ['AGENT_ALIAS_ID'],
    memoryId=os.environ['MEMORY_ID'],
    memoryType='SESSION_SUMMARY',
)
```

### Memory 삭제 (초기화)
```python
runtime.delete_agent_memory(
    agentId=os.environ['AGENT_ID'],
    agentAliasId=os.environ['AGENT_ALIAS_ID'],
    memoryId=os.environ['MEMORY_ID'],
)
```

## System Prompt에서 Memory 활용 지시
```
매 행동 전에 반드시 기억을 확인하세요:
- 이미 방문한 위치는 재방문하지 마세요
- 발견한 코인 위치는 기억에 저장하세요
- 얻은 힌트는 즉시 기억에 저장하세요
```

## 저장할 정보 우선순위
1. **코인 위치** (높은 가치 코인 우선)
2. **장애물 위치** (반복 충돌 방지)
3. **방문한 경로** (재탐색 방지)
4. **게임 힌트** (규칙·보너스 관련)

## 대회 팁
- **세션 간 Memory 유지**: 같은 memoryId 재사용 시 이전 세션 정보 활용 가능
- **Memory 용량 한계**: 요약본 저장 권장 (전체 로그 X)
- **조회 먼저**: invoke_agent 호출 전 get_agent_memory로 현황 파악
