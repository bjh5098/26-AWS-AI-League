# 4.1 Amazon Bedrock AgentCore — 치트시트

## TL;DR
AgentCore = AI 에이전트가 실제로 실행되는 기반 환경. 모델·도구·메모리·가드레일을 연결하는 실행 엔진.

## 핵심 흐름
에이전트 생성 → prepare_agent → 별칭 생성 → invoke_agent

## Key API Calls (boto3)

### 에이전트 생성
```python
import boto3, os

bedrock_agent = boto3.client('bedrock-agent', region_name=os.environ['AWS_REGION'])

response = bedrock_agent.create_agent(
    agentName='ai-league-agent',
    foundationModel='anthropic.claude-3-haiku-20240307-v1:0',  # 속도 우선
    instruction=open('agent/system_prompt.txt').read(),
    agentResourceRoleArn=os.environ['AGENT_ROLE_ARN'],
)
agent_id = response['agent']['agentId']
```

### 에이전트 준비 (액션그룹 등록 후 필수)
```python
bedrock_agent.prepare_agent(agentId=agent_id)
# prepare 완료까지 대기
import time
while True:
    status = bedrock_agent.get_agent(agentId=agent_id)['agent']['agentStatus']
    if status == 'PREPARED': break
    time.sleep(2)
```

### 별칭 생성 (invoke에 필요)
```python
alias = bedrock_agent.create_agent_alias(
    agentId=agent_id,
    agentAliasName='v1'
)
alias_id = alias['agentAlias']['agentAliasId']
```

### 에이전트 호출
```python
runtime = boto3.client('bedrock-agent-runtime', region_name=os.environ['AWS_REGION'])

response = runtime.invoke_agent(
    agentId=os.environ['AGENT_ID'],
    agentAliasId=os.environ['AGENT_ALIAS_ID'],
    sessionId='session-001',
    inputText='지도를 탐색하고 코인을 수집하세요.',
    enableTrace=True,
)

# 스트리밍 응답 파싱
for event in response['completion']:
    if 'chunk' in event:
        print(event['chunk']['bytes'].decode())
```

### 액션그룹(Lambda Tool) 등록
```python
bedrock_agent.create_agent_action_group(
    agentId=agent_id,
    agentVersion='DRAFT',
    actionGroupName='MapTools',
    actionGroupExecutor={'lambda': os.environ['LAMBDA_PATHFINDING_ARN']},
    functionSchema={
        'functions': [
            {
                'name': 'find_path',
                'description': '현재 위치에서 목표 위치까지 최단 경로를 찾습니다.',
                'parameters': {
                    'current_x': {'type': 'integer', 'required': True},
                    'current_y': {'type': 'integer', 'required': True},
                    'target_x': {'type': 'integer', 'required': True},
                    'target_y': {'type': 'integer', 'required': True},
                }
            }
        ]
    }
)
```

## 주요 파라미터
| 파라미터 | 값 예시 | 설명 |
|---------|--------|------|
| foundationModel | anthropic.claude-3-haiku-20240307-v1:0 | 속도 우선 (대회용) |
| foundationModel | anthropic.claude-3-5-sonnet-20241022-v2:0 | 품질 우선 |
| agentStatus | CREATING → NOT_PREPARED → PREPARED | prepare 전까지 invoke 불가 |

## 주요 에러
| 에러 | 원인 | 해결 |
|------|------|------|
| ResourceNotFoundException | alias_id 또는 agent_id 오류 | .env 값 재확인 |
| ConflictException | 에이전트가 PREPARED 상태 아님 | prepare_agent() 재실행 |
| ThrottlingException | 호출 빈도 초과 | 재시도 + 지수 백오프 |

## 대회 팁
- **Haiku 모델 사용**: 응답 속도가 Sonnet 대비 2-3배 빠름 → 제한시간 내 더 많은 행동 가능
- **prepare 상태 확인 필수**: DRAFT 상태에서 invoke하면 에러 발생
- **enableTrace=True**: 에이전트 추론 과정 확인 가능 → 디버깅에 활용
